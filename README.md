## Projeto de Testes - Sistema de Controle de Produtos

Projeto desenvolvido com o objetivo de demonstrar conhecimentos em Qualidade de Software, automação de testes,
web e boas práticas de documentação de testes.

### Objetivos
- Validar funcionalidades da aplicação de Controle de Produtos
- Automatizar cenários críticos utilizando Selenium WebDriver e JUnit 5
- Demonstrar a aplicação do padrão Page Object

### Aplicação testada
A aplicação de Controle de Produtos é uma interface simples desenvolvida em HTML, CSS e JavaScript.
Tem como referência o curso online de Selenium disponível na plataforma Udemy.
Link de acesso: https://www.udemy.com/course/testes-automatizados-curso-completo-com-selenium-e-java/?couponCode=PMNVD2025

### Tecnologias Utilizadas
- Java 21
- Maven (Gerenciamento de Dependências)
- JUnit 5
- Selenium Web Driver 4.33.0
- Chrome Driver

### Documentações de Testes
Os artefatos de QA encontram-se na pasta: 

*docs/*

Contendo:
- Plano de Testes
- Cenários de Testes
- Casos de Testes
- Registro de Bugs
- Evidências de execução dos bugs

A documentação foi incluída no repositório para fins de demonstração profissional e simula artefatos que, em 
ambiente corporativo, normalmente seriam mantidos em ferramentas como Jira, Xray e Zephyr.

### Como executar o Projeto

1. Clonar o repositório

```bash
git clone https://github.com/Gabriela-Reiss/Projeto_Testes_Selenium.git
```

2. Entrar na pasta do projeto

```bash
cd automatizado
```

3. Executar os testes

```bash
mvn test
```
